# 1. Requisitos Funcionais

<p align="justify">A <i>Tabela 1</i> a seguir contém os Requisitos Funcionais (RF) elicitados utizando a técnica de Brainstorm.</p>

| ID   |                                 Requisito                                 | Prioridade | Requisitos Relacionados |
| :--: | :-----------------------------------------------------------------------: | :--------: | :---------: |
| RF01 |             O software deve permitir que o cliente faça o seu próprio cadastro no aplicativo.            |  Alta      |     -       |
| RF02 |             O software deve permitir que o cliente faça seu login no aplicativo.             |  Alta      |     -       |
| RF03 |             O software deve permitir que o cliente selecione o serviço desejado.          |  Alta      |     -       |
| RF04 |             O software deve permitir que o cliente selecione o profissional.          |  Alta      |     -       |
| RF05 |             O software deve permitir que o cliente marque um horário para o serviço desejado.       |  Média     |           |
| RF06 |O software deve permitir que o cliente pague por pix o serviço, dentro do aplicativo.  |  Média     |     RF05    |
| RF07 |     O software deve fornecer um QrCode e uma chave aleatória para o cliente efetuar o pagamento.          |  Alta      |     RF06    |
| RF08 |             O software deve permitir que o cliente selecione se ele gostaria de um lembrete de seu horário marcado na barbearia.            |  Média     |     RF03    |
| RF09 |              O software deve permitir que o cliente selecione a antecedência deste lembrete (dias, horas, minutos).                  |  Média     |     RF03    |
| RF10|             O software deve enviar uma mensagem por Whatsapp do lembrete do agendamento do cliente conforme ele tenha selecionado          |  Alta      |     RF05    |
| RF11|           O software deve permitir que o cliente edite as informações de seu cadastro.       |  Alta      |     RF05    |
| RF12|             O software deve permitir que o cliente consulte os seus agendamentos (ver serviço, dia, horário e barbeiro).          |  Alta      |     RF05    |
| RF13|            O software deve permitir que o cliente saia (Log Out) do aplicativo.          |  Alta      |     RF05    |
| RF14|             O software deve permitir que o cliente escolha o tema de sua preferência (claro/escuro).          |  Alta      |     RF05    |
| RF15|            O software deve permitir que o cliente veja fotos dos serviços disponibilizados pela barbearia na Home em formato de carrossel.         |  Alta      |     RF05    |
| RF16|            O software deve permitir que o cliente edite seu horário previamente marcado (cancelar/remarcar reserva).        |  Alta      |     RF05    |
| RF17|             O software deve permitir que o cliente edite seu horário previamente marcado até 24 horas antes do horário reservado, sendo bloqueada a opção de cancelamento/remarcar depois dessas 24 horas de antecedência.        |  Alta      |     RF05    |
| RF18|            O software deve permitir que o barbeiro consiga ver os horários dos clientes agendados com ele.      |  Alta      |     RF05    |
| RF19|            O software deve permitir que o gestor consiga gerenciar os horários de funcionamento (consegue bloquear horários da agenda para clientes não marcarem).        |  Alta      |     RF05    |
| RF20|          O software deve permitir que o gestor consiga marcar horário para clientes já cadastrados.        |  Alta      |     RF05    |
| RF21|            O software deve permitir que o gestor consiga reservar um horário para um cliente não cadastrado (voltado para pessoas de mais idade).        |  Alta      |     RF05    |
| RF22|            O software deve permitir que o gestor cadastre barbeiros com foto, nome e especialidade.       |  Alta      |     RF05    |
| RF23|            O software deve permitir que o gestor acesse o histórico de agendamentos e preferências do cliente.       |  Alta      |     RF05    |
| RF24|             O software deve permitir que o gestor crie, edite e remova serviços.       |  Alta      |     RF05    |
| RF25|            O software deve permitr que o gestor atualize preços e descrições dos serviços exibidos no aplicativo.        |  Alta      |     RF05    |
| RF26|            O software deve permitir que o gestor altere o status dos clientes (ativo, inativo e viajante).      |  Alta      |     RF05    |
| RF27|            O software deve alterar o status de um cliente para inativo caso ele não tenha marcado nenhum horário dentro de 3 meses, exceto usuários de status "viajante".     |  Alta      |     RF05    |
| RF28|            O software deve mandar uma notificação pelo aplicativo de promoções de cortes para clientes de status inativo.    |  Alta      |     RF05    |
| RF29|            O software deve gerar um relatório de desempenho da barbearia (Mostrando: quanto faturou, despesas, lucro).    |  Alta      |     RF05    |
| RF30|            O software deve gerar um relatório de frequência dos clientes (clientes ativos, inativos).|  Alta      |     RF05    |
